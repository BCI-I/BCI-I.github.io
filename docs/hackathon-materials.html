Preparing for the hackathon

This page contains material to learn about the software and hardware that will be used for the BCI-I hackathon.


Before the hackathon

In order to access some of the software needed you will need:

  • a GitHub account
  • a Docker Hub acount
  • a Duckietown Hub account

Making sure that you have all accounts ready ahead of the competition will help you save time that you can spend hacking instead.

Hardware requirements

To participate in the hackathon, each team needs to be able to stream data from the EEG headset and to send and receive data to and from the Duckiebot. You will need to rely on the manufacturer's software to do so, and this means that you will need devices that can run the software.

The Duckiebot requires Duckiebot software that can only run on Linux and MacOS laptops, and the EEG headset requires X.on software an Android device with Bluetooth connection to stream the data to the laptop. So, we strongly encourage all teams to ensure that they have access to such devices.

We will have a very limited number of devices available for teams that do not have access to one, but we cannot guarantee that we will have enough for everyone.

X.on EEG headset

The X.on EEG headset is an affordable and easy to use EEG headset. It's light and wireless, making it very portable yet it's capable of recording high quality EEG data from its 8 channels (+ reference) and 3-axis accelerometer.


To learn more about the X.on design and specifications, you can check the website for the X.on EEG headset. This video will give you a quick overview of how to setup and use the headset.

X.on Software

The X.on EEG device has Bluetooth connectivity to stream data wirelessly. However, at the moment it cannot be directly connected to a Linux or MacOS laptop. Instead, the data is streamed to an Android device first. Then, the device will send the data to the laptop over WiFi.
For this to work, you will need to install the X.on app on the Android device. On the day of the hackathon we will provide instructions on how to download the `.apk` file and install it on your device. We will also share a brief manual describing how to use the app to connect to the X.on headset.


With the headset connected to the Android device you're ready to start streamining data to a laptop. At this point, you can use any laptop you want since the headset is already connected to the Android device. The only requirements is that both phone and laptop must be connected to the same WiFi network.
We will use Lab Streaming Layer (LSL) to stream the data from the phone to the laptop. The easiest way to setup and use LSL is with the pylsl Python library. We recomend familiarizing yourself with both LSL and pylsl before the hackathon.
With the X.on connected to the Android phone, in the X.on App go to "monitor" - the data will start to be transmitted, ready to receive it in LSL. For streaming the data on your laptop, you can use this code as a starting point: 


      """Example program to show how to read a multi-channel time series from LSL."""

      from pylsl import StreamInlet, resolve_stream
      
      
      def main():
          # first resolve an EEG stream on the lab network
          print("looking for an EEG stream...")
          streams = resolve_stream('type', 'EEG')
      
          # create a new inlet to read from the stream
          inlet = StreamInlet(streams[0])
      
          while True:
              # get a new sample (you can also omit the timestamp part if you're not
              # interested in it)
              sample, timestamp = inlet.pull_sample()
              print(timestamp, sample)
      
      
      if __name__ == '__main__':
          main()
